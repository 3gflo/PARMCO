PARMCO

Phone APP RP4 Motor Control: A complete vertical slice of embedded systems engineering, from Kotlin UI to ARM Assembly.

System Architecture

PARMCO isn't just a motor controller; it's a demonstration of how disparate technologies—Android mobile apps, Bluetooth RFCOMM stacks, Linux C servers, and ARM Assembly—integrate into a single, cohesive feedback loop.

The Journey of a Command

1. The Trigger

User adjusts PWM on the Kotlin Android App. A formatted ASCII string (e.g., RPM:750) is sent via Bluetooth RFCOMM.

2. The Logic

A C Server on the Raspberry Pi receives the packet. It hands off the Target vs. Actual RPM to an ARM Assembly routine for high-speed error calculation.

3. The Action

The L293D H-Bridge drives the 12V motor. An IR Sensor counts blade rotations, sending live telemetry back to the phone at 4Hz.

The Tech Stack

Kotlin C (BlueZ) ARM Assembly Raspberry Pi 4 Bluetooth RFCOMM

Core Assembly Logic (control.S)

adjust_pwm:
    sub r1, r1, r0      // Calculate Error (Target - Current)
    cmp r1, #1500
    blt gentle_gain
    asr r1, r1, #4      // Aggressive Gain for high speeds
    b apply_pwm
